Future Butcher

Do you remember Drug Wars? I spent hours during high school math class playing that game on my TI-83 calculator. I got hooked on the game's simplicity, and how RNG and a somewhat forbidden subject combined to make it infinitely replayable.

So in 2017 when I needed a project to learn the Elixir language, Drug Wars came to mind. I had started reading the excellent Pragmatic Programmers book Functional Web Development with Elixir, OTP, and Phoenix, which provides an in-depth tutorial building a Battleship-type game. But instead of following the material to the letter, I decided to build my own game with the book as a guide. I didn't want to rip off John E. Dell's original either, so I spent some time brainstorming a new world.

Design

Right off the bat, Future Butcher had a cyberpunk feel. The original name was Slinger, and the player could buy and sell brain implants with different gameplay effects. But the concept was too unfamiliar for players who hadn't read any William Gibson. So instead I took the game in a new direction - cannibalism.

I kept the cyberpunk feel, with a UI intended to resemble an old terminal. Some of the events and encounters underscore that feel, with mentions of predatory corporations, weak (and possibly failed) governments, and near anarchy.

Architecture

Following the PragProg book's approach, I designed a stateful game engine in Elixir. It handles turns, player statistics, game rules, etc. The engine is a dependency of the server application, built with Phoenix.

The application use's Phoenix Channels to communicate with the client, and OTP supervised processes to spin up a state machine when the game starts. Authentication is non-existent, because I wanted to provide an arcade experience.

The original (ugly) client was an Emberjs application, but in 2022 I replaced it with React. This client is much cleaner, snappier, and less buggy. Both client versions communicate to the server via Websockets.

William Storck Portfolio

Design

Because the goal of the site is to showcase the painter's art, I chose a minimalist style that wouldn't intefere. Mr. Storck and I worked together on a unique color palette intended to mimic the frames commonly used with his work.

Architecture

This is a simple Elixir/Phoenix CRUD application. However to allow Mr. Storck to manage his paintings and collections I created a gated admin site. This was a great opportunity to dive into Phoenix Contexts.

Unfortunately the portfolio is now offline.

About Me

Hello, I'm Jon Choukroun - a software engineer, (sort-of) musician, chess enthusiast, and film buff. I currently work at Apple, on the iCloud Mail team, where I focus on the web client. This means I mostly use React with Typescript.

I started my career as a sound engineer, recording and mixing albums and concert films. I briefly composed music for tv ads, but soon left the music industry. I don't get much of a chance to play drums or compose anymore, but I occasionally still play guitar.

After a boring stint in management consulting, I taught myself to code. I've been a programmer since 2016, building web applications with Ruby, Elixir, Node, and React. But my favorite language to work in is C++. Before Apple I worked for Twenty20, a platform for stock photos. Unfortunately the startup was acquired and later shuttered. Now, in addition to my Mail web client work at Apple, I enjoy messing around on various projects.
